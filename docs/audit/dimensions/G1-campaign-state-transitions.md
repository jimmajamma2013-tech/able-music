# Dimension G1 — Campaign State Transitions
**Category:** Core Product Logic & User Flows
**Phase:** 1 (HIGHEST PRIORITY — do before visual)

All four campaign states (profile / pre-release / live / gig) must transition correctly and predictably. Two separate state machines exist: `computeState()` in able-v8.html (fan-facing) and `computeEffectiveState()` + `computeAutoState()` in admin.html (dashboard). They must agree. Any divergence means fans see a different state than the artist configured.
