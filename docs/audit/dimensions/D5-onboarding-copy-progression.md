# Dimension D5 — Onboarding Copy Progression
**Category:** Copy, Voice & Messaging
**Phase:** 4 (Copy)
**Status:** Not started

The onboarding wizard (start.html) is the first conversation ABLE has with an artist. It must feel like movement toward something real — not filling in a form. Each step should feel like a natural question a music-savvy friend would ask, not a data-entry field with a label. Step labels, hints, progress copy, placeholder text, confirmation copy, and error states all must use the ABLE voice. No SaaS micro-copy, no "complete your profile", no progress bars that say "You're 80% done!" Full compliance means every string in start.html passes the artist-depth test: would a thoughtful independent musician feel respected reading this?
