# Dimension G4 — Gig Mode Expiry
**Category:** Core Product Logic & User Flows
**Phase:** 1 (Foundation)
**Status:** Not started

Gig mode is a 24-hour manual override that puts ticket links front-and-centre and shows an "on tonight" tag. It must auto-expire after 24 hours and return the profile to the correct previous state — not to the hardcoded `'profile'` state, but to whatever state the campaign auto-logic would compute based on `releaseDate`. This dimension audits: the expiry write on activation, the expiry check on every read, the countdown timer in admin, the cleanup of `stateOverride` after expiry, and the return-to-correct-state logic. Full compliance means gig mode expires reliably, leaves no orphaned localStorage state, and the fan-facing page and admin dashboard agree on the post-gig state within one page load.
